# Upgrade 16: Preparing for Interop protocol upgrade

This page provides external-facing information about the Fusaka readiness upgrade for OP Stack chains. It summarizes scope, timelines, required actions, and resources for chain operators and partners.

This is NOT Fusaka adoption on L2. Itâ€™s a readiness upgrade to make the Optimism protocol compatible with L1 that activates the Fusaka fork. L2 adoption will happen in a future upgrade.

<Callout type="info">
All of the tasks below must be done as soon as the OP Stack Fusaka readiness release is available and before Fusaka activates on L1 (October 14th for Sepolia chains).
</Callout>

## For chain operators

* Update `proxyd` to v4.19.0 or greater
* Update `op-batcher` to v1.16.0-rc.1
* Update node software


### For permissionless fault proof enabled chains

Chains running permissionless fault proofs will need to:
* Update `op-challenger` to use the new prestates. The new prestate can be found in [the Superchain Registry](https://github.com/ethereum-optimism/superchain-registry/blob/main/validation/standard/standard-prestates.toml)
* Deploy new dispute game contracts with the new prestate
