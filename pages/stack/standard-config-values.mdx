# Standard Configuration Values

This section outlines the recommended standard configuration values for deploying
OP Stack software. These values ensure consistency, security, and optimal
performance across different deployments. The parameters are divided into four
main categories:

## Consensus Parameters
  * Description: Fixed or changeable parameters that affect the blockchain's properties.
  * Examples: Block size, gas limit, etc.

## Policy Parameters
   * Description: Changeable parameters that do not affect consensus.
   * Examples: Transaction fees, governance rules, etc.

## Admin Roles
  * Description: Roles responsible for managing upgrades and protocol changes.
  * Examples: Protocol upgrades, security patches, etc.

## Service Roles
  * Description: Roles managing daily operations of the chain.
  * Examples: Node operations, monitoring, etc.

For detailed standard configuration values, refer to the
[OP Stack Configurability](https://specs.optimism.io/protocol/configurability.html) page.
